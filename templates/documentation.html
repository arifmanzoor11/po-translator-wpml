PO File Translator Documentation

Overview

The PO File Translator is a web-based tool that automatically translates WordPress/Gettext PO (Portable Object) files from one language to another using machine translation.

Features

  • Drag & Drop Interface - Simply drag your PO file onto the upload area
  • Multiple Languages - Support for 18+ languages
  • Auto-Detection - Automatically detect the source language
  • Real-time Progress - Live translation progress
  • Batch Processing - Efficiently handles multiple strings
  • Download Ready - Translated PO file ready to use

Supported Languages

Source Languages

Auto Detect (recommended), English, French, Spanish, German, Italian, Portuguese, Dutch, Russian, Chinese, Japanese, Korean, Arabic, Hindi, Turkish, Polish, Vietnamese, Thai

Target Languages

All languages listed above except Auto Detect

How to Use

Step 1: Select Languages

  1. Choose your Source Language (or leave as "Auto Detect")
  2. Select your Target Language

Step 2: Upload File

  • Drag & Drop or use "Choose File"
  • Requirements: .po file, max 10MB, valid format

Step 3: Start Translation

  1. Click "Start Translation"
  2. Monitor progress
  3. Download starts automatically

File Format Requirements

Supported Format

  • Extension: .po
  • Encoding: UTF-8 recommended
  • Structure: Standard Gettext format

Example PO File Structure


#: template.php:123
msgid "Hello World"
msgstr ""

#: template.php:456
msgid "Welcome to our website"
msgstr ""

After Translation


#: template.php:123
msgid "Hello World"
msgstr "Bonjour le monde"

#: template.php:456
msgid "Welcome to our website"
msgstr "Bienvenue sur notre site web"

Translation Process

What Gets Translated

  • msgid strings
  • Comments and references
  • Context formatting preserved

What Stays the Same

  • File headers
  • Line numbers
  • Formatting

Best Practices

Before Translation

  • Backup original PO file
  • Validate format
  • Ensure UTF-8 encoding

After Translation

  • Review translation output
  • Test in application
  • Generate .mo file

Troubleshooting

Common Issues

  • Translation Errors: Re-check source language and internet
  • Slow Processing: Large files or server load

API Integration

Upload Endpoint


POST /upload
Content-Type: multipart/form-data

Parameters:
- file: PO file
- source_lang: Source language code
- target_lang: Target language code

Progress Tracking


GET /translation-progress
Content-Type: text/event-stream

Returns: Server-sent events with progress updates

WordPress Integration

  1. Upload .po file to /wp-content/languages/
  2. Generate .mo file using Poedit
  3. Activate language in WordPress

File Naming Convention
